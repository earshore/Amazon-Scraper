用户选择"最小改动:代理+UA轮换"方案绕开 Amazon 反爬(Playwright/PA-API/第三方API为备选)。

  - `undici` 的 `ProxyAgent` 作为 per-request `dispatcher` 路由住宅代理。
  - 代理来源优先级:`AMAZON_PROXIES`(逗号分隔轮换) > `AMAZON_PROXY` > `HTTPS_PROXY`/`HTTP_PROXY`。
  - 5 组真实 Chrome UA,每组带匹配的 `Sec-CH-UA` 客户端提示(避免 UA/提示不一致这一典型 bot 特征)。

  - 重试:`SCRAPE_MAX_RETRIES`(默认4),抖动退避1.2–3s;命中 robot-check/5xx 重试,404 不重试。
- `web/server/index.ts`:改用 `fetchAmazonPage`,新增 `dotenv/config` 加载;health 回报 `proxyConfigured`/`maxRetries`;423 文案加 proxy 提示。
- 依赖加 `undici` + `dotenv`;因 vite8/plugin-react peer 告警需 `npm install --legacy-peer-deps`。
- 新增 `web/.env.example`、`web/.gitignore`。README 增补代理配置章节与安装提示。

## Amazon Scraper Web — 多 ASIN 并行抓取

用户需求:支持多 ASIN 并行抓取。

实现(解析层 `scraper.ts` 完全未动):
- `web/server/index.ts` 的 `POST /api/scrape` 改为批量:接受 `asins`(数组或逗号/空格分隔字符串)或旧 `asin`(单)。
  - 去重、上限 `SCRAPE_MAX_BATCH`(默认50,超出→400)、逐个校验(非法→failed product)。
  - 并行抓取用 `mapWithConcurrency`(保留顺序),并发 `SCRAPE_CONCURRENCY`(默认5;单 ASIN 强制1)。
  - 每项独立成功/失败:失败条目以 `scrape_status:"failed"`+`error` 进入 `products[]`,不影响其它项。
  - 聚合 `metadata` 增加 `requested_asins/succeeded/failed`;单 ASIN 且 robot-check 时保留旧 423 契约,否则统一 200(部分失败也在 200 内)。
  - 批量模式每项重试 `min(SCRAPE_MAX_RETRIES,2)` 控制总时延;无全局 AbortController(避免部分丢失)。
- `web/src/ScraperPage.tsx`:输入框改多行 `textarea`(多行/逗号/空格分隔),结果区改多卡片列表(每张卡一个 ASIN,成功/失败统一渲染)+ 批量成功/失败计数 + 导出(文件名含数量)。新增 `ProductCard` 子组件。
- `web/src/index.css`:新增 `.asin-textarea/.asin-hint/.batch-summary/.results-list/.count-ok/.count-bad` 样式。
- README 增补批量用法、响应结构、新环境变量表。

验证:空 ASIN→400;51 个→400(超上限);3 个(含 1 非法)→200 且 products 聚合正确(metadata requested_asins:3/succeeded:0/failed:3);
单 ASIN 404→200 failed product(仅 robot-check 才 423)。`tsc -b && vite build` 通过。
